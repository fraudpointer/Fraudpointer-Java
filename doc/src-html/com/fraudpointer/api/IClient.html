001    package com.fraudpointer.api;
002    
003    import com.fraudpointer.api.models.AssessmentSession;
004    import com.fraudpointer.api.models.Event;
005    import com.fraudpointer.api.models.FraudAssessment;
006    
007    /**
008     * IClient is the interface that each java client should implement in order to be used as a client proxy to access
009     * FraudPointer Service. </b>Important Note: </b> java client jar comes with a ready to use implementation of this service. You need to use
010     * ClientFactory to instantiate this implementation and start using FraudPointer Service.<br/>
011     * <br/>
012     * As a user of the FraudPointer Client Library you should declare a variable of IClient type and then use ClientFactory.construct() to get an
013     * instance of a class that implements this interface.<br/>
014     * <br/>
015     * <span id='sample_code'>Below, you can see a sample client that uses IClient to communicate with FraudPointer Server:</span><br/>
016     * <br/>
017     *
018     * <iframe src='{@docRoot}/src-html/com/fraudpointer/ConsoleClientExample.html' width="100%" height="350px"></iframe>
019     *
020     */
021    public interface IClient {
022        /**
023         * Communicates with FraudPointer Server and gets back a ready to use AssessmentSession object.This is the first method that you should call. <br/>
024         * <br/>
025         * It should be used as soon as you want to start using the FraudPointer service. Without an AssessmentSession object
026         * you cannot do anything.<br/>
027         * <br/>
028         * @return An object of type AssessmentSession as returned by FraudPointer Server. Object should contain the AssessmentSession.id
029         * that uniquely identifies this AssessmentSession. You should use this AssessmentSession.id in whichever place the
030         * AssessmentSession id is required.<br/>
031         * <br/>
032         * Method always returns a valid object, unless an exception is thrown.
033         * @throws ClientException It may throw an ClientException if an error occurs
034         */
035        AssessmentSession createAssessmentSession () throws ClientException;
036    
037        /**
038         * Appends an Event to an AssessmentSession object. <br/>
039         * <br/>
040         * This method also communicates with the FraudPointer Server, which will store all Event data alongside with the AssessmentSession
041         * given.<br/>
042         * <br/>
043         * You can call this method as many times as you want (as long as you pass a new Event on each call).
044         * It is actually recommended to do so. In other words, it is recommended to create various Events, of
045         * various types and send them to FraudPointer to be stored against an AssessmentSession.
046         * <br/>
047         *
048         * @param assessmentSession Should be constructed with a call to createAssessmentSession()
049         * @param paramEvent Should be constructed with a call to new Event(event_type)
050         * @return An object of type Event as returned by the FraudPointer Server.<br/>
051         * <br/>
052         * Method always returns a valid object, unless an exception is thrown.
053         * @throws ClientException It may throw a ClientException if an error occurs
054         *
055         */
056        Event appendEventToAssessmentSession (AssessmentSession assessmentSession, Event paramEvent) throws ClientException;
057    
058        /**
059         * This method should be called in order to evaluate the fraud level of an AssessmentSession. <br/>
060         * <br/>
061         * This method should be called after having sent to FraudPointer Server one or more Events. It is used
062         * to evaluate the fraud risk of the AssessmentSession. Note that you can ask multiple interim 
063         * Fraud Assessments, but you can only ask for one final (non-interim) Fraud Assessment. Interim Fraud Assessments
064         * can be requested in a cycle of interactions with the FraudPointer Server such as the following:<br/>
065         * <br/>
066         * <ul>
067         * <li> create assessment session </li>
068         * <li> create event  </li>
069         * <li> add data to event  </li>
070         * <li> add data to event  </li>
071         * <li> ...  </li>
072         * <li> add data to event  </li>
073         * <li> append event to assessment session  </li>
074         * <li> create another event </li>
075         * <li> add data to new event </li>
076         * <li> add data to new event </li>
077         * <li> ...  </li>
078         * <li> add data to new event  </li>
079         * <li> append new event to assessment session </li>
080         * <li> create interim fraud assessment </li>
081         * <li> create one more event </li>
082         * <li> add data to last created event </li>
083         * <li> add data to last created event </li>
084         * <li> ... </li>
085         * <li> add data to last created event </li>
086         * <li> append last created event to assessment session </li>
087         * <li> create interim fraud assessment </li>
088         * <li> .... </li>
089         * <li> create final (non<li>interim) fraud assessment </li>
090         * </ul>
091         * <br/>
092         * The last and only final (non-interim) fraud assessment will also create a CASE in FraudPointer Application.
093         * <br/>
094         * @param assessmentSession A valid AssessmentSession, previously created with createAssessmentSession()
095         * @param interim A boolean value true or false. If true, then an interim FraudAssessment will be created.
096         * If false, a final (non-interim) FraudAssessment will be created
097         * @return A FraudAssessment valid object filled in with information returned by FraudPointer Server. It contains the Fraud
098         * Assessment Result.
099         * @throws ClientException It may throw a ClientException if an error occurs
100         */
101        FraudAssessment createFraudAssessment (AssessmentSession assessmentSession, boolean interim) throws ClientException;
102    
103        /**
104         * Gets the results of a previously assessed session. <br/>
105         * <br/>
106         * Use this method to retrieve a previously created Fraud assessment.
107         * When you create a fraud assessment with createFraudAssessment you receive
108         * the result of the assessment and its Id. Using the session id and the assessment id you can retrieve that result
109         * with this method.<br/>
110         * <br/>
111         * @param assessmentSession A valid AssessmentSession, previously created with createAssessmentSession()
112         * @param assesmentId A valid assessment id previously obtained with createFraudAssessment()
113         * @return A FraudAssessment valid object filled in with information that is related to this Fraud Assessment like Profile and Case
114         * @throws ClientException It may throw a ClientException if an error occurs
115         */
116        FraudAssessment getFraudAssessment (AssessmentSession assessmentSession, String assesmentId) throws ClientException;
117    
118        /**
119         * Use this method to generate a hash of a credit card number. Use the generated hash to send the encrypted credit card number
120         * to FraudPointer Server instead of the credit card number itself. <br/>
121         * <br/>
122         * FraudPointer Server tries to identify the existence of the same credit card number in various transactions. These
123         * transactions either take place during the same session or take place in different sessions, but at the same time, or
124         * took place in a session in the past. However, FraudPointer Server does not want to store the credit card numbers in clear format
125         * and it does not need to do that in order to accomplish its goal. Hence, you need to encrypt them using the method provided here.
126         * Note that hash is one-way encryption method and FraudPointer Server cannot derive the credit card number from the hash.
127         * <br/>
128         * This method does not communicate with the FraudPointer Server to generate the hash. Works locally.<br/>
129         * <br/>
130         * Also, note that there is a sample console application that only encrypts a credit card. The class is called EncryptCreditCard
131         * and you can see the code here: <a href='{@docRoot}/src-html/com/fraudpointer/EncryptCreditCard.html'>EncryptCreditCard</a>. You can
132         * quickly run this utility from the command line with the following command too:<br/>
133         * <br/>
134         * <code>java -cp fraudpointer_client.jar com.fraudpointer.EncryptCreditCard <a_credit_card_number_to_encrypt></code><br/>
135         * <br/>
136         * and it will print for you the string that will be sent to Fraudpointer server if that particular card is used in a transaction.
137         * <br/>
138         * @param creditCardNumber The credit card number that you want to get its hash value.
139         * @return The hash of the credit card number. Note that if an empty string is returned, then something has gone wrong during the encryption.
140         * in that case, method throws a ClientException.
141         */
142        String creditCardHash (String creditCardNumber) throws ClientException;
143    
144    } // interface IClient
